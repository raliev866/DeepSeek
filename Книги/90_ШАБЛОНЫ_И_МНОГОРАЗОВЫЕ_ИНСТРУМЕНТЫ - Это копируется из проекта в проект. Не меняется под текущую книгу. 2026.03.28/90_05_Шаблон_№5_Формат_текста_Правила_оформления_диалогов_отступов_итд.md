# 90_05_ШАБЛОН-ФОРМАТА-ТЕКСТА.md

## Правила оформления

### 1. Заголовки
- `#` — только для названия книги (1 раз)
- `##` — для названий томов
- `###` — для названий разделов
- `####` — для названий глав
- `#####` — для подразделов внутри главы

### 2. Отступы и структура
- Между заголовком и текстом — пустая строка
- Между абзацами — пустая строка
- Внутри абзаца — без переносов (один логический блок)

### 3. Выделения
- **Жирный** — для ключевых терминов при первом упоминании
- *Курсив* — для акцентов, примеров, кейсов
- `Код` — для формул, единиц измерения, значений pH

### 4. Списки
- Маркированные списки: `-` или `*`
- Нумерованные списки: `1.`, `2.`, `3.`

### 5. Цитаты и важные блоки
> **Важно!** — для ключевых утверждений
> **Пример:** — для иллюстраций
> **Исследования:** — для ссылок на научные данные

### 6. Таблицы
Используются для сравнения, систематизации данных.
Формат: стандартный Markdown.

### 7. Вопросы и задания
В конце каждой главы — блок `❓ Вопросы и задания`

### 8. Связи между главами
В конце главы — блок `🔗 Связь с другими главами` с таблицей

### 9. Резюме
В конце главы — блок `📌 Краткое резюме` с 3-5 выводами
```

## Мой анализ

**Что мне нравится:**

1.  **Чёткая иерархия заголовков** — это важно для навигации по книге. В вашем репозитории сейчас эта иерархия часто нарушается (например, название главы идёт `#`, а не `####`).
2.  **Выделения по смыслу** — жирный для терминов, курсив для примеров — это хорошая практика.
3.  **Блоки с иконками** (`> **Важно!**`, `> **Пример:**`) — визуально выделяют ключевые элементы.
4.  **Единые блоки в конце** — вопросы, связи, резюме — это соответствует моему Шаблону №1.

**Что можно улучшить или дополнить:**

1.  **Отсутствует единый стиль для иконок** — в разных главах используются разные эмодзи (🔬, 🩺, 📖, ❓, 🔗, 📌). Нужно утвердить один набор.
2.  **Нет правил для списков** — когда использовать маркированный, а когда нумерованный.
3.  **Нет правил для таблиц** — формат заголовков, выравнивание.
4.  **Нет правил для сносок и ссылок** — как оформлять литературу.
5.  **Нет правила для «Кейса из практики»** — в моём шаблоне есть этот блок, но в текущем шаблоне формата он не описан.

Я предлагаю **взять ваш шаблон за основу** и дополнить его несколькими пунктами для полной стандартизации. Вот **расширенная версия**:

```markdown
# 90_05_ШАБЛОН-ФОРМАТА-ТЕКСТА.md

## Правила оформления текста книги

### 1. Иерархия заголовков

| Уровень | Элемент | Пример |
|---------|---------|--------|
| `#` | Название книги | `# Кислотно-щелочное равновесие. Научный подход к здоровью` |
| `#` | Том | `# Том 1. Основы физиологии и регуляции` |
| `##` | Раздел внутри тома | `## Раздел 2. Буферные системы организма` |
| `###` | Название главы | `### Глава 3. Бикарбонатная буферная система` |
| `####` | Подраздел внутри главы | `#### 3.1. Химическая основа бикарбонатной системы` |

**Важно:** Между заголовком и следующим текстом — всегда пустая строка.

---

### 2. Структура главы (по Шаблону №1)

Каждая глава следует единой структуре (см. `Шаблон №1. Структура главы`).

---

### 3. Отступы и форматирование

| Элемент | Правило |
|---------|---------|
| Абзацы | Разделяются пустой строкой |
| Внутри абзаца | Нет принудительных переносов |
| Списки | Элементы списка — с новой строки |
| Цитаты | Символ `>` в начале строки |

---

### 4. Выделения и их значение

| Выделение | Когда использовать | Пример |
|-----------|-------------------|--------|
| **Жирный** | Ключевые термины при первом упоминании, важные утверждения | **бикарбонатная буферная система** |
| *Курсив* | Акценты, примеры из жизни, кейсы, метафоры | *Представьте, что организм — это...* |
| `Код` | Числовые значения, единицы измерения, формулы | `pH 7.35–7.45`, `HCO₃⁻`, `pCO₂` |

---

### 5. Блоки с иконками (визуальное выделение)

Используются в начале абзаца или как отдельный блок:

| Иконка | Назначение | Формат |
|--------|------------|--------|
| `> **Важно!**` | Ключевое утверждение, которое нельзя пропустить | Отдельный блок с пустыми строками до и после |
| `> **Пример:**` | Иллюстрация, клинический кейс | Обычно внутри абзаца или как краткий блок |
| `> **Исследования:**` | Ссылка на научные данные | С указанием источника |
| `> **Чек-лист:**` | Список для самопроверки | Маркированный список внутри блока |

---

### 6. Списки

| Тип | Когда использовать | Формат |
|-----|-------------------|--------|
| Маркированный (`-`) | Перечисление без порядка, чек-листы | `- пункт 1` |
| Нумерованный (`1.`) | Последовательность шагов, иерархия | `1. первый шаг` |

**Правила:**
- После маркера — пробел
- Элементы списка — с новой строки
- Между списком и текстом — пустая строка

---

### 7. Таблицы

| Правило | Пример |
|---------|--------|
| Заголовок таблицы — жирный | `| Параметр | Значение |` |
| Выравнивание — по умолчанию влево | Можно использовать `:---` для левого, `---:` для правого |
| После таблицы — пустая строка | |

**Пример:**
```markdown
| Параметр | Норма | Отклонение |
|----------|-------|------------|
| pH крови | 7.35–7.45 | < 7.35 — ацидоз |
```
