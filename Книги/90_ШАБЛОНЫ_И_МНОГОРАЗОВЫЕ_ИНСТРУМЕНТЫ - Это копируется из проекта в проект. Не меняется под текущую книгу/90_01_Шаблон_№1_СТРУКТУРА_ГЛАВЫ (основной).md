## Пояснения к доработкам

### 1. Навигационный блок (Том, Раздел, связи)

Добавлен в начало главы:

```markdown
**📍 Навигация:**
- **Том:** [Номер]. [Название тома]
- **Раздел:** [Номер]. [Название раздела] *(если есть)*
- **Предыдущая глава:** [Название] → [ссылка или указание]
- **Следующая глава:** [Название] → [ссылка или указание]
```

**Зачем:**
- Читатель всегда знает, где он находится в структуре книги.
- Упрощает навигацию, особенно если читать отдельные главы.
- Соответствует лучшим практикам технической документации и сложных非художественных книг.

### 2. Аудиодружественные версии таблиц

Используется конструкция:

```html
<!-- АУДИОВЕРСИЯ: начало -->
<details>
<summary>📢 Текстовая версия для аудио (нажмите, чтобы развернуть)</summary>

*Для прослушивания:*
[текстовая версия таблицы]

</details>
<!-- АУДИОВЕРСИЯ: конец -->
```

**Как это работает:**

| Формат | Поведение |
|--------|----------|
| **Markdown / GitHub** | Отображается как складной блок. По умолчанию скрыт, пользователь может раскрыть. |
| **DocX / Word** | При конвертации через Pandoc или другой инструмент можно настроить фильтр, чтобы удалять блоки с `<!-- АУДИОВЕРСИЯ: ... -->`. |
| **HTML / PDF** | Аналогично DocX — можно удалить при постобработке. |
| **Аудиокнига** | При подготовке аудиофайлов вы (или скрипт) можете извлечь только текст внутри `<details>`, удалив визуальные таблицы. |

**Варианты для будущей автоматизации:**

1. **Для текстовых версий (PDF, DOCX, HTML):**  
   Использовать скрипт, который удаляет всё между `<!-- АУДИОВЕРСИЯ: начало -->` и `<!-- АУДИОВЕРСИЯ: конец -->`, оставляя только визуальные таблицы.

2. **Для аудиоверсий:**  
   Использовать скрипт, который удаляет визуальные таблицы и оставляет только текстовые версии внутри `<details>`.

## Обновлённый контрольный список для проверки главы

- [ ] **Навигационный блок** есть в начале (Том, Раздел, связи)
- [ ] Иерархия заголовков соответствует правилам
- [ ] Есть блок `Ключевая идея главы` в начале
- [ ] Есть блок `Введение: почему это важно`
- [ ] Основная часть разбита на подразделы
- [ ] Есть блок `Клиническое значение` или `Практические рекомендации`
- [ ] Для каждой таблицы есть **аудиодружественная версия** внутри `<details>`
- [ ] Есть блок `Вопросы и задания` (минимум 3)
- [ ] Есть блок `Связь с другими главами` (таблица + аудиоверсия)
- [ ] Есть блок `Краткое резюме` (3–5 пунктов)
- [ ] Есть блок `Литература` (с аудиоверсией)
- [ ] Использованы единые иконки
- [ ] Выделения терминов и значений по правилам
