# ШАБЛОН ФОРМАТА ТЕКСТА
## Единый стандарт оформления книг серии «Кислотно-щелочное равновесие»

---

## 1. ОБЩАЯ СТРУКТУРА ТЕКСТА

### 1.1. Иерархия заголовков

```
# Название книги (Уровень 1) — только в начале файла
# Название раздела (Уровень 1)
## Глава [номер]. Название главы (Уровень 2)
### [номер подглавы]. Название подглавы (Уровень 3)
#### Второстепенный блок (Уровень 4 — использовать редко)
```

**Правила:**
- **Уровень 1 (#)** — название книги (только в начале файла) и названия разделов
- **Уровень 2 (##)** — названия глав (Глава 1, Глава 2...)
- **Уровень 3 (###)** — подглавы (1.1, 1.2, 1.3...)
- **Уровень 4 (####)** — только для сложных глав, где нужно дробить глубже (1.1.1, 1.1.2...)

**Пример:**
```
# Кислотно-щелочное равновесие: Желудочно-кишечный тракт — ключ к системному здоровью

# Раздел I. Желудок: кислотный барьер организма

## Глава 1. Физиология желудочного сока

### 1.1. Состав желудочного сока
### 1.2. Париетальные клетки
### 1.3. Протонная помпа
```

## 2. ОФОРМЛЕНИЕ ТЕКСТА

### 2.1. Выделения

| Элемент | Оформление | Пример |
|---------|------------|--------|
| **Ключевые термины** | `**жирный**` | Соляная кислота (**HCl**) вырабатывается... |
| *Акцент, иностранные слова* | `*курсив*` | Этот процесс называется *апоптозом*... |
| ***Особо важное*** | `***жирный курсив***` | ***Запомните: без HCl невозможно усвоение B12*** |
| `Код / команды` | \`обратные кавычки\` | Введите команду `генератор` |

**Правила:**
- **Жирный шрифт** — только для ключевых терминов при первом упоминании
- *Курсив* — для иностранных слов, названий, акцентов
- ***Жирный курсив*** — только для критически важных выводов (1-2 раза на главу)

### 2.2. Списки

**Маркированный список (основной):**
```
- Первый пункт
- Второй пункт
  - Подпункт (с отступом 2 пробела)
  - Ещё подпункт
- Третий пункт
```

**Нумерованный список (последовательность):**
```
1. Первый шаг
2. Второй шаг
   1. Подшаг 2.1
   2. Подшаг 2.2
3. Третий шаг
```

**Правила:**
- После маркера списка (`-` или `1.`) — **один пробел**
- Между пунктами списка — **пустая строка не нужна** (кроме сложных случаев)
- Если пункт длинный (больше 3 строк) — разделяй на подпункты

### 2.3. Цитаты и примеры

```
> Это цитата из исследования или важное утверждение.
> Она выделяется таким блоком.

> **Важно:** Если внутри цитаты нужно выделение — используй **жирный**.
```

### 2.4. Чек-листы (специальный формат)

В конце каждой главы — **обязательный чек-лист**:

```
## ✅ Чек-лист: что запомнить из главы

- [ ] Пункт первый для самопроверки
- [ ] Пункт второй для самопроверки  
- [ ] Пункт третий для самопроверки

> **На заметку:** Если есть важное дополнение к чек-листу
```

**Почему так:**
- Чек-лист — единственное завершение главы (никаких «выводов» или «резюме»)
- Читатель видит, что усвоил, а что нужно повторить

## 3. ОФОРМЛЕНИЕ ТАБЛИЦ (ОСОБЫЙ СЛУЧАЙ)

### 3.1. Общее правило

**Таблицы — только в электронной версии.** Для аудиоформата у каждой таблицы должен быть **текстовый дубль**.

### 3.2. Формат таблицы в Markdown

```
| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|-------------|-------------|-------------|
| Ячейка 1.1  | Ячейка 1.2  | Ячейка 1.3  |
| Ячейка 2.1  | Ячейка 2.2  | Ячейка 2.3  |
```

### 3.3. Текстовый дубль таблицы (для аудио)

Сразу после таблицы идёт **текстовый дубль** в таком формате:

```
📋 **Текстовая версия таблицы для прослушивания:**

Таблица [Название] содержит следующие данные:
- По строке 1: [пояснение]
- По строке 2: [пояснение]
- Ключевой вывод из таблицы: [вывод]
```

**Пример:**
```
📋 **Текстовая версия таблицы для прослушивания:**

Таблица «Время гибели патогенов при pH 2,0» показывает:
- Холерный вибрион погибает за 5–10 минут
- Сальмонелла — за 10–20 минут
- Стафилококк — за 20–40 минут

Главный вывод: даже кратковременное снижение кислотности делает организм уязвимым для кишечных инфекций.
```

## 4. РАБОТА С ТЕРМИНАМИ

### 4.1. Первое упоминание термина

При первом упоминании сложного термина:
1. Выдели его **жирным**
2. Дай краткое объяснение в скобках или через тире

**Пример:**
> Главную роль играют **париетальные клетки** (обкладочные клетки желудка, вырабатывающие соляную кислоту).

### 4.2. Иностранные термины

При первом упоминании давай оригинал в скобках:
> Синдром избыточного бактериального роста (**СИБР**, от англ. *SIBR — Small Intestinal Bacterial Overgrowth*)

### 4.3. Аббревиатуры

При первом упоминании — полностью, потом можно сокращать:
> Соляная кислота (**HCl**) играет ключевую роль. Далее мы будем говорить просто HCl.

## 7. ТРЕБОВАНИЯ К АБЗАЦАМ

### 7.1. Длина абзаца

- **Идеально:** 3–5 предложений
- **Максимум:** 7–8 предложений (если тема сложная и требует развития)
- **Минимум:** 1 предложение (для акцента или связки)

### 7.2. Структура абзаца

1. **Первое предложение** — тема абзаца (что сейчас будем обсуждать)
2. **Середина** — развитие мысли, аргументы, примеры
3. **Последнее предложение** — вывод или переход к следующей мысли

**Пример:**
> **Ключевую роль в производстве HCl играют париетальные клетки.** Эти клетки находятся в слизистой оболочке тела желудка и дна желудка. В них происходит удивительный процесс: из углекислого газа и воды образуется угольная кислота, которая распадается на ион водорода и бикарбонат. Ион водорода с помощью протонной помпы перекачивается в просвет желудка в обмен на ион калия. **Так создаётся та самая кислая среда, необходимая для пищеварения.**

### 7.3. Связки между абзацами

Используй слова-связки:
- «Однако...»
- «С другой стороны...»
- «Важно понимать...»
- «Рассмотрим подробнее...»
- «Отсюда следует вывод...»

## 10. ПРИМЕР ИДЕАЛЬНОЙ ГЛАВЫ (ЭТАЛОН)

```
# Раздел I. Желудок: кислотный барьер организма

## Глава 4. Гипохлоргидрия (пониженная кислотность желудка)

### 4.1. Что это такое?

**Гипохлоргидрия** — это состояние, при котором желудок вырабатывает недостаточно соляной кислоты (HCl). В лёгких случаях говорят о пониженной кислотности, в тяжёлых — об ахлоргидрии (полном отсутствии HCl).

💡 **Знаете ли вы?** После 40 лет уровень кислотности снижается у каждого второго человека, а после 60 — у 70–80%.

### 4.2. Как распознать?

Симптомы гипохлоргидрии часто путают с повышенной кислотностью. Вот основные признаки:

- Тяжесть после еды, особенно мясной
- Отрыжка с тухлым запахом
- Вздутие через 1–2 часа после еды
- Чувство, что еда «стоит камнем»
- Непереносимость мяса (хочется есть, но после приёма тяжело)
- Хронические запоры или диарея

> ⚠️ **Важно:** Изжога тоже может быть симптомом низкой кислотности! Когда привратник плохо закрывается, содержимое забрасывается в пищевод и вызывает жжение.

### 4.3. Почему это опасно?

Недостаток HCl ведёт к цепочке проблем:

1. Белки не расщепляются полностью
2. Не активируется пепсин
3. Не поступает сигнал поджелудочной железе
4. Желчь застаивается
5. В тонкий кишечник попадают непереваренные частицы и патогены

**Результат:** СИБР, дефициты витаминов (особенно B12 и железа), хроническое воспаление.

---

## ✅ Чек-лист: что запомнить из главы 4

- [ ] Гипохлоргидрия — это пониженная кислотность желудка
- [ ] После 40 лет риск гипохлоргидрии резко возрастает
- [ ] Симптомы: тяжесть после еды, отрыжка, вздутие
- [ ] Изжога может быть признаком низкой кислотности
- [ ] Недостаток HCl ведёт к СИБР и дефициту витаминов
```

## 4. РАБОТА С ТАБЛИЦАМИ

**4.0. По мере возможности делай всё чтобы вообще не составлять таблицы, а ищи заменители таблиц в обычном тексте.** Но если вынужденно приходится таблицы, тогда читай следующие пункты.

**4.1. Оформление таблицы**

```
Таблица [номер]. [Название]

<table>
  <thead>
    <tr style="background-color: #d9e1ec; font-weight: bold;">
      <th>Заголовок 1</th>
      <th>Заголовок 2</th>
    </tr>
  </thead>
  <tbody>
    <tr style="background-color: #f2f2f2;">
      <td>Строка 1, ячейка 1</td>
      <td>Строка 1, ячейка 2</td>
    </tr>
    <tr style="background-color: #ffffff;">
      <td>Строка 2, ячейка 1</td>
      <td>Строка 2, ячейка 2</td>
    </tr>
  </tbody>
</table>
```

**Цветовая схема:**
- **Шапка таблицы:** `#d9e1ec` — тёмно-серый с голубым оттенком, выделяется.
- **Чётные строки:** `#f2f2f2` — светло-серый.
- **Нечётные строки:** `#ffffff` — белый.

**4.2. Текстовый дубль для аудиоверсии**

```
[Название] (без слова «Таблица», без номера)

**Название строки 1.** Развёрнутое, связное описание первой строки. Объединяет информацию из всех колонок. Оформлено как законченный рассказ. Без отсылок к таблице.

**Название строки 2.** Аналогичное описание второй строки.
```

**Назначение дубля:**
- В **печатной версии** остаются **и таблица, и текст** (оба блока).
- В **аудиоверсии** **таблица удаляется**, текстовый дубль остаётся и звучит полноценно.

## 5. ЯЗЫКОВЫЕ ТРЕБОВАНИЯ

**5.1. Чистота языка**
Если мы пишем на **русском языке** — текст содержит **только русские слова**.  
Никаких:
- ❌ иностранных слов, англицизмов,
- ❌ транслитераций,
- ❌ латиницы,
- ❌ вставок на других языках.

**Исключение:** устоявшиеся термины, не имеющие русского аналога (например, «Интернет», «Wi-Fi» — но и для них по возможности ищем замену).

**5.2. Адаптация под другие языки**
Если мы пишем на **азербайджанском** — только азербайджанские слова.  
Если на **английском** — только английские.  
Принцип един для всех языков: **текст должен быть чистым, без чужеродных вкраплений**.

**5.3. Контроль**
Я проверяю **каждое своё сообщение** на наличие иностранных слов перед отправкой.

# ШАБЛОН ОФОРМЛЕНИЯ ТАБЛИЦ С ТЕКСТОВЫМ ДУБЛЕМ

*Инструкция для всех будущих сессий*

---

## 1. ОБЩАЯ СТРУКТУРА БЛОКА

```
Таблица [номер]. [Название таблицы]

| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|-------------|-------------|-------------|
| Ячейка 1.1  | Ячейка 1.2  | Ячейка 1.3  |
| Ячейка 2.1  | Ячейка 2.2  | Ячейка 2.3  |

[Название таблицы] (без слова «Таблица», без номера)

**Название строки 1.** Здесь идёт полное, развёрнутое, законченное предложение, которое объясняет смысл всей первой строки таблицы. Объём — от одного до трёх предложений. Информация берётся из всех колонок строки и связывается в читаемый текст.

**Название строки 2.** Аналогичное развёрнутое описание второй строки. Никаких отсылок к таблице («в столбце 2», «как показано выше»). Текст должен быть самодостаточным.
```

---

## 2. ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА

### 2.1. Заголовки
- **Перед таблицей:** слово **«Таблица»** + порядковый номер + точка + название таблицы.  
  *Пример:* `Таблица 1. Режимы работы ИИ`
- **Перед текстовым блоком:** только **название таблицы**, без слова «Таблица» и без номера.  
  *Пример:* `Режимы работы ИИ`

### 2.2. Оформление таблиц (Markdown с HTML-стилями)
В Markdown нет встроенной поддержки цветных строк.  
Чтобы таблицы корректно отображались на всех платформах, мы используем **HTML-разметку внутри Markdown**.

**Шаблон оформления таблицы с чередованием строк:**

```html
Таблица 1. Режимы работы ИИ

<table>
  <thead>
    <tr style="background-color: #d9e1ec; font-weight: bold;">
      <th>Режим</th>
      <th>Когда применять</th>
      <th>Результат</th>
    </tr>
  </thead>
  <tbody>
    <tr style="background-color: #f2f2f2;">
      <td>Архитектор</td>
      <td>Нужно создать структуру, оглавление, карту знаний</td>
      <td>Иерархический план, таксономия, схема</td>
    </tr>
    <tr style="background-color: #ffffff;">
      <td>Фонтан</td>
      <td>Нужны идеи, мозговой штурм, снятие ступора</td>
      <td>Список гипотез, ассоциаций, вариантов</td>
    </tr>
    <tr style="background-color: #f2f2f2;">
      <td>Редактор</td>
      <td>Есть черновик, требуется шлифовка</td>
      <td>Исправленный текст, улучшенный стиль</td>
    </tr>
  </tbody>
</table>
```

**Параметры цветов:**
- **Заглавная строка (`<thead>`):** `#d9e1ec` — тёмно-серый с голубоватым оттенком, выделяется.
- **Чётные строки:** `#f2f2f2` — светло-серый.
- **Нечётные строки:** `#ffffff` — белый.

### 2.3. Текстовый дубль
- Начинается с **повтора названия таблицы** (без слова «Таблица», без номера).
- Каждая строка таблицы превращается в **отдельный смысловой блок**.
- Блок открывается **названием строки**, выделенным **жирным шрифтом**.
- Далее следует **полное, связное предложение (или несколько)**, которое:
  - Объединяет информацию из **всех колонок** данной строки;
  - Излагает её **линейно, повествовательно**, как рассказ;
  - **Не содержит отсылок к таблице** («в левом столбце», «как указано выше», «см. ячейку»);
  - **Не копирует механически** ячейки через запятую, а перерабатывает в человеческую речь;
  - Использует **законченные речевые конструкции** («Этот режим предназначен для...», «Результатом его работы будет...»).

### 2.4. Запреты
- ❌ Никаких «В столбце 2 указано...».
- ❌ Никаких «Как видно из таблицы...».
- ❌ Никаких «Ячейка 1.1 свидетельствует о...».
- ❌ Никаких перечислений ячеек через запятую без грамматической связки.
- ❌ Никаких иностранных слов в русском тексте.

**Режимы работы ИИ**

**Режим «Архитектор».** Этот режим предназначен для создания структуры. Если вам нужно разработать оглавление книги, построить карту знаний, спроектировать иерархию разделов или таксономию понятий — вы включаете режим «Архитектор». Результатом его работы будет чёткий иерархический план, схема или классификация.

**Режим «Фонтан».** Этот режим предназначен для генерации идей. Когда вы в ступоре, когда нужен мозговой штурм, когда требуется список гипотез, вариантов или ассоциаций — вы включаете режим «Фонтан». Результатом его работы будет поток сырых, неотфильтрованных, иногда противоречивых идей, из которых вы потом отбираете нужное.

**Режим «Редактор».** Этот режим предназначен для работы с уже существующим текстом. Вы присылаете черновик — режим «Редактор» его шлифует: исправляет стиль, устраняет повторы, улучшает читаемость, сокращает лишнее. Результатом его работы будет чистый, хорошо написанный текст, готовый к публикации.

**Режим «Критик».** Этот режим предназначен для поиска уязвимостей. Вы даёте проект, текст или стратегию — режим «Критик» находит слабые места, риски, логические противоречия, неудачные формулировки. Результатом его работы будет список того, что нужно исправить, чтобы сделать продукт устойчивее и убедительнее.

## 4. ЧТО ДЕЛАТЬ, ЕСЛИ ТАБЛИЦА БОЛЬШАЯ (НАПРИМЕР, 20 СТРОК)

Правила не меняются:
- Каждая строка таблицы → отдельный текстовый блок с **жирным названием**.
- Никаких пропусков, сокращений, отсылок к «вышесказанному».
- Если строки логически группируются — можно добавить **подзаголовок третьего уровня (`###`)** перед группой, но сам принцип остаётся.

## 5. КОНТРОЛЬНЫЙ ЧЕК-ЛИСТ (ДЛЯ МЕНЯ)

Перед отправкой каждого блока «таблица + текст» я проверяю себя:

- [ ] Перед таблицей стоит **«Таблица N. Название»**?
- [ ] Перед текстовым блоком стоит **только название** (без «Таблицы» и номера)?
- [ ] Таблица свёрстана **HTML-кодом** с указанными цветами фона?
- [ ] Заглавная строка имеет цвет `#d9e1ec`?
- [ ] Строки чередуются: `#f2f2f2` (светло-серая), `#ffffff` (белая)?
- [ ] Каждая строка таблицы превращена в **отдельный текстовый блок**?
- [ ] В текстовых блоках **нет отсылок к таблице**?
- [ ] Текст в блоках — это **связные предложения**, а не копии ячеек?
- [ ] В русском тексте **нет иностранных слов** (кроме устоявшихся терминов)?
- [ ] Для аудиоверсии достаточно удалить **только HTML-таблицу**, и текст останется полностью читаемым?
